# 自動改善バックログ

このリストは、毎日1つだけAIが取り組むための小さな改善候補です。1PRにつき1テーマに限定し、テストが通った場合だけPRにします。

## 安全ルール

- 本番ブランチへ直接pushしない。
- `supabase_schema.sql`、本番データ、RLS、認証、課金、secretsは自動変更しない。
- DB削除、データ削除、破壊的マイグレーションは自動対象外にする。
- 変更は小さく、レビューしやすい範囲にする。
- 失敗したらPRを作らず、GitHub Actionsのログで確認する。

## UI改善

- [ ] 学習カードの余白と文字サイズをスマホで読みやすく微調整する。
- [ ] 学習カード内の品詞・カテゴリ・難易度の表示順を見やすく整える。
- [x] ボタン文言を短くし、スマホで折り返しにくくする。
- [ ] 正解・不正解メッセージの表示を少し分かりやすくする。
- [ ] 入力欄のプレースホルダーを学習中の文脈に合わせて改善する。
- [ ] 復習画面の統計表示を読みやすくする。

## 学習体験改善

- [ ] 新規追加単語が学習画面に出る理由をUI上で短く示す。
- [ ] 苦手数の説明を画面上で自然に補足する。
- [x] 学習カードで「次のカード」後に状態が分かりやすい表示を追加する。
- [ ] 筆記問題の不正解時に、正解単語を見やすく表示する。
- [ ] 穴埋め問題の blank 表示をスマホで見やすくする。
- [ ] 発音再生ボタンのラベルをさらに自然にする。

## テスト追加

- [ ] 単語の正規化処理のテストケースを増やす。
- [ ] 苦手優先と新規単語混在ロジックのテストケースを増やす。
- [ ] 穴埋め問題の大文字小文字対応テストを増やす。
- [ ] AI生成結果のバリデーション関数に対するテストを追加する。
- [ ] CSV保存に必要な列が欠けた場合の補完テストを追加する。

## リファクタ

- [ ] UI文言を小さな定数に分け、変更しやすくする。
- [ ] 学習順序の計算ロジックを読みやすい補助関数に分ける。
- [ ] Supabase接続判定の周辺を小さく整理する。
- [ ] AI追加画面のフォーム作成部分を小さな関数に分ける。
- [x] 学習統計更新処理の重複を減らす。

## 軽微なバグ修正

- [ ] 空の例文で穴埋め問題を開いた場合の表示を分かりやすくする。
- [x] 品詞が未設定の単語が混ざった時の表示ゆれを抑える。
- [ ] 難易度が空欄の単語を読み込んだ時の表示を安定させる。
- [ ] 発音メモが空欄のカード表示を自然にする。

## 自動対象外

次の変更は人間が明示的に依頼した時だけ扱います。

- Supabaseテーブル削除、列削除、データ削除
- RLSポリシー、認証、権限設計の大きな変更
- Streamlit CloudやSupabaseのsecrets変更
- 課金、APIキー、service role keyの取り扱い変更
- 本番データを直接書き換える処理
